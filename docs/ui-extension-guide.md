# Расширение публичного интерфейса

## Что уже есть в проекте

UI-слой выделен в отдельное приложение `apps.ui`.

Он отвечает за:
- глобальный стиль сайта;
- пользовательскую тему (`system / light / dark`);
- язык интерфейса;
- инъекцию UI-состояния в шаблоны.

Ключевые элементы:
- `apps/ui/registry.py` — реестр глобальных стилей;
- `apps/ui/models.py` — конфигурация публичного интерфейса;
- `apps/ui/services.py` — сборка итогового UI-state;
- `apps/ui/middleware.py` — применение пользовательских предпочтений;
- `apps/ui/context_processors.py` — передача UI-state в шаблоны.

## Как работает глобальный стиль

Логика разделена на два уровня:

### 1. Глобальный стиль сайта
Выбирается администратором в `Конфигурации публичного интерфейса`.

Примеры уже существующих стилей:
- `academic`
- `liquid_glass`

### 2. Пользовательская тема
Пользователь может выбрать:
- `system`
- `light`
- `dark`

То есть стиль определяет **визуальный язык сайта**, а тема — **светлый/тёмный режим внутри выбранного языка стиля**.

## Как добавить новый стиль интерфейса

### Шаг 1. Создать CSS-файл
Положите новый файл в:

```text
static/css/ui/styles/
```

Например:

```text
static/css/ui/styles/my-style.css
```

### Шаг 2. Зарегистрировать стиль в реестре
В `apps/ui/registry.py` добавьте:

```python
register_style(
    UiStyleDefinition(
        identifier="my_style",
        label=_("Мой стиль"),
        description=_("Краткое описание стиля."),
        stylesheet_path="css/ui/styles/my-style.css",
        preview_hint=_("Когда этот стиль уместен."),
    )
)
```

### Шаг 3. Использовать переменные базового слоя
Новый стиль лучше строить не с нуля, а поверх переменных из `static/css/ui/base.css`.

Обычно имеет смысл переопределять:
- фоновые поверхности;
- бордеры;
- акцентный цвет;
- карточки;
- стили заголовков;
- навигацию;
- кнопки.

### Шаг 4. Выбрать стиль в админке
После регистрации стиль автоматически появится в модели конфигурации публичного интерфейса.

## Как добавить новый язык

### Шаг 1. Добавить язык в `LANGUAGES`
В `config/settings/base.py`:

```python
LANGUAGES = [
    ("ru", _("Русский")),
    ("en", _("Английский")),
    ("de", _("Немецкий")),
]
```

### Шаг 2. Создать каталог локали

```text
locale/de/LC_MESSAGES/
```

### Шаг 3. Добавить переводы
Нужны файлы:
- `django.po`
- `django.mo`

### Шаг 4. При необходимости добавить локализованные названия сайта
В конфигурации интерфейса можно хранить:
- локализованные версии названия сайта;
- локализованные версии подзаголовка.

## Что доступно в шаблонах

В шаблонах доступны:
- `ui_state`
- `ui_style`
- `ui_theme_mode_choices`
- `ui_language_choices`

Этого достаточно, чтобы постепенно переводить страницы на общий UI-механизм без ручной сборки состояния в каждом view.

## Рекомендации по расширению UI

### Делать
- добавлять новые стили через реестр;
- строить стиль поверх общей системы CSS-переменных;
- использовать общие карточки, кнопки и layout-паттерны;
- тестировать светлую и тёмную тему вместе;
- следить за адаптивностью с самого начала.

### Не делать
- не переопределять глобально HTML-элементы слишком широко (`section`, `article`, `select`, `button`) без namespace;
- не смешивать стили публичного сайта с админкой Django и Wagtail;
- не внедрять новый стиль напрямую в шаблон без регистрации в `apps.ui.registry`.

## Как расширять публичную навигацию

Публичная навигация живёт в основном шаблоне и собирает ссылки как на core-часть прототипа, так и на CMS-раздел.

Если появляется новый самостоятельный раздел:
1. добавьте маршрут;
2. добавьте ссылку в nav / footer;
3. убедитесь, что он соответствует текущей композиции сайта;
4. при необходимости предусмотрите перевод подписи и иконографику.
