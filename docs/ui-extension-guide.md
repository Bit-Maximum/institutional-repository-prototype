# Расширение UI-слоя прототипа

## Что уже подготовлено

В проекте выделен отдельный слой `apps.ui`, который отвечает за:

- глобальный стиль публичного интерфейса;
- пользовательское переключение светлой / тёмной / системной темы;
- пользовательское переключение языка интерфейса;
- внедрение UI-состояния в шаблоны через middleware и context processor.

## Как добавить новый глобальный стиль

1. Создать CSS-файл в `static/css/ui/styles/`.
2. Зарегистрировать стиль в `apps/ui/registry.py` через `register_style(...)`.
3. Использовать CSS-переменные, уже определённые в `static/css/ui/base.css`.
4. После регистрации стиль автоматически появится в административной модели `Конфигурация публичного интерфейса`.

Пример: 

```python
register_style(
    UiStyleDefinition(
        identifier="my_style",
        label=_("Мой стиль"),
        description=_("Описание стиля"),
        stylesheet_path="css/ui/styles/my-style.css",
        preview_hint=_("Когда этот стиль уместен"),
    )
)
```

## Как добавить новый язык

1. Добавить язык в `LANGUAGES` в `config/settings/base.py`.
2. Создать каталог `locale/<code>/LC_MESSAGES/`.
3. Добавить `django.po` и `django.mo`.
4. После этого язык автоматически появится в селекторе интерфейса.

## Как использовать UI-состояние в шаблонах

В любом шаблоне доступны:

- `ui_state` — итоговое состояние интерфейса;
- `ui_style` — активный глобальный стиль;
- `ui_theme_mode_choices` — варианты темы;
- `ui_language_choices` — варианты языков.

Это позволяет постепенно переделывать страницы без изменения базового механизма переключения стилей и языков.
