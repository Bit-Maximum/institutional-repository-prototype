# Работа редактора: издания, черновики, коллекции, превью

## Редактор издания

Страница добавления издания и страница редактирования черновика используют общий редакторский интерфейс.

Его задачи:
- сократить визуальный шум;
- показать текущее состояние записи;
- дать возможность публиковать материал поэтапно.

## Как устроен редактор

Редактор разделён на секции:
1. Источник
2. Ключевые метаданные
3. Участники и связи
4. Издательские сведения
5. Дополнительные данные

Справа показывается панель состояния:
- статус записи;
- прогресс заполнения;
- основные действия;
- краткая памятка.

## Прогресс публикации

Система считает готовность по набору ключевых полей:
- название;
- файл или внешняя ссылка;
- год издания;
- язык;
- подтип издания;
- авторы;
- аннотация / описание.

Черновик можно сохранять поэтапно.
Публикация требует заполнения этого минимума.

## Работа со справочниками прямо из редактора

Для полей вида:
- язык;
- периодичность;
- авторы;
- научные руководители;
- ключевые слова;
- места публикации;
- издатели;
- копирайты;
- библиографические описания;
- графические издания;

используются улучшенные dropdown-контролы:
- можно искать по мере ввода;
- можно выбирать несколько значений;
- для части полей можно создать новый элемент прямо из редактора.

Это позволяет не уходить в Django admin ради каждого нового справочного значения.

## Файл, внешняя ссылка и извлечение текста

Редактор может:
- загрузить файл;
- указать внешнюю ссылку;
- использовать оба варианта, если это необходимо.

Если формат поддерживается, система попытается извлечь текст и использовать его для:
- поиска;
- аннотаций;
- предзаполнения некоторых полей.

Если извлечь текст нельзя, запись всё равно может существовать и участвовать в поиске по метаданным.

## Превью издания

### Варианты превью
1. **Ручная загрузка** — редактор сам выбирает изображение.
2. **Автогенерация** — из PDF, изображения или fallback-обложки.

### Приоритет
Если редактор загрузил превью вручную, оно имеет приоритет над автогенерацией.

## Черновики

Черновики нужны для поэтапной работы над изданием:
- запись можно сохранить до полной готовности;
- позже вернуться к ней;
- доработать метаданные, превью и связи;
- затем опубликовать.

Список черновиков — это рабочая зона редактора, а не публичная часть сайта.

## Коллекции

### Что умеют коллекции
- формировать пользовательские подборки;
- показывать layered-preview из верхних публикаций;
- отображать краткие метаданные и рейтинг;
- делиться ссылкой на коллекцию.

### Каталог коллекций
Карточка коллекции показывает:
- превью-стек;
- название;
- краткое описание;
- автора;
- дату обновления;
- количество изданий;
- итоговую оценку.

### Страница коллекции
На странице коллекции:
- есть лаконичный верхний блок с метаданными;
- показаны карточки публикаций;
- у карточек есть краткий фрагмент описания;
- можно скопировать share-ссылку.

## Кто может редактировать издания

В текущем прототипе редактирование публикаций через сайт ограничено административными ролями:
- `is_staff`
- `is_admin`
- `is_superuser`

Обычный пользователь не может редактировать чужие публикации.
